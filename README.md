## About the Book

This book is not about prompt shortcuts. It is about building engineering judgment while working with AI — and about recognizing when AI is wrong, even when it sounds completely right.

The book develops two frameworks:

- **GUARD** — a defensive posture for analysis problems (those with a single correct answer), covering geometry preparation, error uncovering, assumption exposure, reverse prompting, and divide and conquer
- **SPAR** — a proactive posture for synthesis problems (open-ended design), covering sampling the design space, posing questions, anchoring decisions, and reviewing and reconciling

Both frameworks are discipline-independent and are developed through worked examples drawn from mechanical, electrical, chemical, and civil engineering.

## The Reference Card

The GUARD/SPAR reference card is the primary resource in this repository. It is designed for two uses:

**As a student reference** — a scannable guide to consult while working on AI-assisted engineering problems. GUARD is structured as a checklist; SPAR as a vocabulary of moves.

**As an AI prompt block** — each framework section includes a collapsible prompt block that can be copied directly into an AI chat session (Claude, ChatGPT, Gemini, etc.) to instruct the AI to apply the framework automatically.

The card is intentionally a companion to the book, not a substitute for it. The frameworks make full sense only with the worked examples that develop them across the book's chapters.
